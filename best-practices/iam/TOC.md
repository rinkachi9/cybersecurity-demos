# IAM Security Best Practices: Table of Contents

This directory contains a structured framework for managing identities and access within Google Cloud Platform (GCP). These guidelines follow the **Principle of Least Privilege (PoLP)** and **Zero Trust** security models.

## 📋 Security Practices List

1.  **[01-principle-of-least-privilege.md](./01-principle-of-least-privilege.md)**: Giving only the minimum access needed for a specific task.
2.  **[02-group-based-management.md](./02-group-based-management.md)**: Scaling access control through security groups.
3.  **[03-identity-federation-sso.md](./03-identity-federation-sso.md)**: Centralizing identity management with SSO and WIF.
4.  **[04-service-account-management.md](./04-service-account-management.md)**: Securing non-human identities and avoiding static keys.
5.  **[05-conditional-access-context-aware.md](./05-conditional-access-context-aware.md)**: Enforcing access based on context (location, time, device).
6.  **[06-regular-access-reviews-auditing.md](./06-regular-access-reviews-auditing.md)**: Continuous monitoring and auditing of IAM policies.
